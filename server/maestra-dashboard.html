What is Maestra? A centralized system built by Jordan Snyder during her time as VP of Platform for all three of Meow Wolf's permanent installations: open source MIT. It manages all the devices in your installation. Imagine you have 4 projectors, 2 audio zones, and 3 sensor stations across a venue â€” Maestra lets every device register itself, sync its state (brightness, color, scene), advertise media streams (NDI, Syphon, Spout), and receive commands via REST, WebSocket, or MQTT â€” all from one dashboard. Each device runs a tiny client (TouchDesigner, Max/MSP, Arduino, web browser) that talks to the Maestra server. The TouchDesigner TOX wraps the Maestra API so you can drop it into any TD project and connect in seconds (in theory).   What each layer does:

  Scope           Runs locally. Takes a video input, applies AI diffusion models
                  (LongLive, StreamDiffusion, Krea Realtime), outputs transformed video.
                  Outputs via NDI or screen capture. No network coordination.

  TouchDesigner   Receives Scope output via NDI In TOP. Composites, adds effects,
                  maps to projector outputs. Runs the Maestra TOX.

  Maestra TOX     Registers this machine with the fleet. Syncs state
                  (scene, brightness, color, prompts) from the server.
                  Advertises its NDI stream so other machines can discover it.

  Maestra Server  Central coordinator. One state change propagates to all
                  connected entities. Monitors heartbeats. Maintains stream registry.
01
Scope Generates Locally
Each machine runs Scope with its own GPU. Camera or media in, AI-transformed video out. Scope outputs via NDI â€” TouchDesigner picks it up with an NDI In TOP. Scope doesn't know about other machines. It doesn't need to.
02
Maestra Coordinates the Fleet
The TOX in each TD instance registers with the Maestra server. When you push a state change â€” scene: 3, color: [0.8, 0.2, 1.0] â€” every connected machine receives it. TD reads the new state and updates Scope's prompt, switches LoRAs, or adjusts post-processing.
03
Streams Stay Discoverable
Each machine advertises its NDI output through Maestra's stream registry. A monitor station can discover all streams on the network â€” no manual IP hunting. Heartbeats tell you instantly if a machine drops.
# Example: Maestra state change triggers Scope prompt update via OSC

# In the Maestra TOX's par_exec callback:
def onValueChange(par, prev):
    if par.name == 'Scene':
        scene = par.eval()
        prompts = {
            0: 'ethereal crystal cave, bioluminescent',
            1: 'baroque gold cathedral, dramatic lighting',
            2: 'neon cyberpunk rain, reflective surfaces',
        }
        # Send new prompt to Scope's local API or write to shared file
        op('scope_osc_out').sendOSC('/scope/prompt', prompts.get(scene, ''))
